# 国际化（i18n）

插件可以为其显示文本（名称、描述、参数标签等）提供翻译。现在推荐直接在 `manifest.json` 中内联声明，也继续兼容旧的 `locales/*.json` 目录方式。

## 推荐方式：在 `manifest.json` 中内联 locales

对于 `name`、`description`、`category`、`label`、`group` 等字段，先写 i18n key，再在 `manifest.json` 中声明各语言词典：

```json title="manifest.json"
{
  "name": "meta.name",
  "description": "meta.description",
  "category": "meta.category",
  "params": [
    {
      "key": "speed",
      "label": "params.speed",
      "group": "params.groups.animation"
    }
  ],
  "locales": {
    "en-US": {
      "meta": {
        "name": "Rainbow",
        "description": "Flowing rainbow animation across all LEDs",
        "category": "Animation"
      },
      "params": {
        "speed": "Speed",
        "groups": {
          "animation": "Animation"
        }
      }
    },
    "zh-CN": {
      "meta": {
        "name": "彩虹",
        "description": "流动的彩虹动画效果",
        "category": "动画"
      },
      "params": {
        "speed": "速度",
        "groups": {
          "animation": "动画"
        }
      }
    }
  }
}
```

:::tip
推荐字段名是 `locales`。为了兼容旧写法，`i18n` 和 `translations` 也同样可用。
:::

## 旧方式：`locales/` 目录

如果你想保持旧插件结构，或需要兼容旧打包方式，依然可以把翻译文件放在插件的 `locales/` 目录中：

```
plugins/effect.my_effect/
├── manifest.json
├── main.lua
└── locales/
    ├── en-US.json
    └── zh-CN.json
```

## 使用 i18n 键

在 `manifest.json` 中，使用引用本地化词典条目的键，代替字面字符串：

```json title="manifest.json"
{
  "name": "meta.name",
  "description": "meta.description",
  "category": "meta.category",
  "params": [
    {
      "key": "speed",
      "label": "params.speed",
      "group": "params.groups.animation"
    }
  ]
}
```

## Locale 文件

像 `meta.name` 这样的键，既可以解析扁平字典，也可以解析嵌套对象。下面示例使用嵌套 JSON，因为它更直观，也与内联 `locales` 的写法一致：

在 `locales/` 目录下创建按 locale 代码命名的 JSON 文件：

```json title="locales/en-US.json"
{
  "meta": {
    "name": "Rainbow",
    "description": "Flowing rainbow animation across all LEDs",
    "category": "Animation"
  },
  "params": {
    "speed": "Speed",
    "preset": "Preset",
    "colors": "Colors",
    "groups": {
      "animation": "Animation"
    }
  }
}
```

```json title="locales/zh-CN.json"
{
  "meta": {
    "name": "彩虹",
    "description": "流动的彩虹动画效果",
    "category": "动画"
  },
  "params": {
    "speed": "速度",
    "preset": "预设",
    "colors": "颜色",
    "groups": {
      "animation": "动画"
    }
  }
}
```

## 解析流程

1. Core 先加载旧的 `locales/*.json` 文件。
2. 然后再把 `manifest.json` 中 `locales` / `i18n` / `translations` 的内容合并覆盖上去。
3. 如果同一个语言下存在相同键，`manifest.json` 中的值优先。
4. Core 再读取 manifest 字段值（例如 `"meta.name"`），并在合并后的词典中解析。
5. 如果找不到匹配项，则直接使用原始字符串。

这意味着你可以混合使用字面字符串和 i18n 键：

```json
{
  "name": "meta.name",
  "publisher": "Skydimo"
}
```

这里 `name` 使用 i18n 查找，而 `publisher` 保持原样。

## 支持的语言

Skydimo 目前支持：

| 代码 | 语言 |
|------|------|
| `en-US` | English |
| `zh-CN` | 简体中文 |

## LocalizedText 格式（API）

当 Core 通过 WebSocket API 发送本地化文本时，使用 `LocalizedText` 格式：

```json
{
  "raw": "Rainbow",
  "byLocale": {
    "en-US": "Rainbow",
    "zh-CN": "彩虹"
  }
}
```

前端根据用户当前语言设置，使用 `resolveLocalizedText()` 解析此格式。
